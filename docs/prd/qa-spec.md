# QA Specification

Version: 2.0
Status: Normative

## 1) Test Layers

- Unit tests: scoring/state transitions and invariants
- E2E tests: critical user journeys in browser runtime
- Manual QA: milestone hardware and install validation

## 2) Required Local Gate

- Command: `pnpm run complete-check`
- Local gate MUST execute full required suite before merge intent.
- Local E2E MUST cover Chromium + WebKit profiles.

## 3) Unit Test Policy

- Unit tests MUST cover scoring engine and state transitions comprehensively.
- Coverage threshold (global):
  - lines >= 80%
  - branches >= 80%

## 4) E2E Policy

- Required CI E2E runs MUST use mobile viewport profiles.
- Required CI browser engines: Chromium + WebKit.
- E2E retry policy: single retry for failures.

## 5) Required Critical E2E Scenarios

- Setup and start match
- Score progression and game/set updates
- Advantage and Golden Point behavior
- Tiebreak and deciding-set behavior
- Undo correctness
- Match end and Continue Playing
- Reset with confirmation
- Persistence resume/discard prompt
- Speech mute behavior and announcement triggering
- Offline app-shell availability for full lifecycle

## 6) Docs-Only PR Exception

If PR changes only `docs/**` and markdown meta files:
- Required checks reduce to docs/lint validation
- Unit and E2E checks are not required for merge

Detection is path-based automation.

## 7) Manual Milestone QA

Milestones are flagged manually by owner.

At each flagged feature milestone, task owner MUST perform:
- Full manual regression for presenter remote flows
- Validation on at least one known-good presenter device
- PWA install validation on target mobile device

Manual results MUST be recorded in the linked project task item.

## 8) CI Artifacts

- Test artifacts retention: 7 days
- Include reports/screenshots/videos as generated by CI jobs
